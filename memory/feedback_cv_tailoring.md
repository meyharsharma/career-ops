Every application must have a tailored CV and cover letter. Not just the summary — the whole thing.

**Why:** User wants each application to feel customized, not generic. Generic resumes get filtered.

**How to apply:**
1. **Bullet format:** Use Google's X-Y-Z: "Accomplished [X] as measured by [Y], by doing [Z]". Lead with impact.
2. **Summary:** Rewrite for each role — lead with the most relevant framing (AI Engineer vs ML Engineer vs Data Scientist)
3. **Skills order:** List skills most relevant to the JD first
4. **Bullet order:** Within each job, put the most JD-relevant bullet first
5. **Cover letter:** When a form has a cover letter/additional info textarea, always generate a customized 3-4 paragraph letter:
   - Para 1: What draws me to this company (reference their product)
   - Para 2: Most relevant experience mapped to JD
   - Para 3: What I'd bring on day one
   - Para 4: One line close
6. **Reference file:** `data/lessons.md` has Q&A patterns for common form questions
